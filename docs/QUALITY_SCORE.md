# Quality Score

Use this scorecard to grade the quality of major repo areas over time.

## Record

- Owner: repository maintainers
- Review date: YYYY-MM-DD
- Scope: architecture, docs, validation, security, product clarity

## Rubric

Score each area from 1 to 5.

- `1`: missing or misleading
- `2`: partial and fragile
- `3`: usable with clear gaps
- `4`: strong and mostly current
- `5`: exemplary and mechanically enforced

## Expectations

- Every scored area needs concrete evidence.
- Any score below `4` should include action items.
- Action items should point to a plan, doc update, or code change.
